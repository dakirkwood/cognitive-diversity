# Plan Mode

This is a post-commitment discussion. The decision to build has been
made. The central question is: **"How do we execute this well?"**

## Arbiter Behavioral Overrides

- **Scope expansion is constrained.** If a perspective raises a
  concern that questions whether the work should happen at all, log
  it to the Parking Lot. Do not debate it. The commitment has been
  made.

- **The Innovator is constrained to execution.** The Innovator may
  propose better approaches to building what's committed — different
  architecture, alternative tools, creative implementation patterns.
  The Innovator may not reopen the decision to build or propose
  replacing the committed work with something else.

- **"Needs Owner Input" applies to execution questions.** What
  technology to use, which tradeoff to accept, how to sequence the
  work. Not whether the work should happen.

- **Agenda framing should be implementation-focused.** Subtopics
  should stress-test execution: "What's the right deployment strategy
  for this service?" not "Should we build this service?"
